Mini tutoriel Angular 10

Quelques informations

Angular est tout d’abord apparu en 2010 pour la première fois, avant de changer de nom pour AngularJS. Il va falloir par la suite attendre jusqu’à 2016 pour qu’une version Angular 2 apparaisse en 2016. Développé par Google, il permet la création de Single Page Applications et d’application web.

Création de projet

Afin de créer votre premier projet Angular il vous faudra un Node.js, Npm et Git :
Node.js : C’est un moteur Javascript permettant de faire le service, la création et le regroupement d’application et bien plus.
Npm : C’est un gestionnaire qui est de base inclut dans le fichier Node.js. Il permet de télécharger et d’installer les bibliothèques nécessaires pour utiliser l’application web.
Git : C’est un client permettant de faire le lien entre votre projet et un système de contrôle de version distribué. Il peut-être utile si vous souhaitez par exemple upload votre site sur github.
Pour commencer à l’utiliser créer un dossier (« espace de travail ») dans lequel vous comptez mettre tous vos projet Angular. Lance git dans ce dossier puis entrer :
npm install
Pour installer et configure le serveur Json entrer :
Npm i g json-server
Npm I faker
Utiliser la commande npm audit fix en cas d’erreur lors de l’installation

Quelques commandes pour vous aider à créer votre premier projet

Ng n [nom] – pour créer votre projet
Ng n [nom] –-routing -pour créer un projet utilisant un système de route
Npm install bootstrap – installer bootstrap
Ng g c [nom du composant] -pour creer un composant il faut le faire dans le bon dossier
Ng g s [nom du service] -pour générer un service
Ng g c [nom de la classe] pour créer une classe
Code . -lancer votre projet dans visual studio
Npm start – lancer votre projet

Mini tutoriel Crud

L'acronyme C.R.U.D signifie en anglais Create, Read, Update et Delete. Ceux sont les 4 opérations de base que doivent pouvoir effectuer un Crud. On utilise le CRUD en créant un service GenericService qui prend comme paramètres un Objet T et un ID. Cela permet d’utiliser ce service pour faire ces quatre opérations sur n’importe quel objet créer. Cela peut être un avion, un client, des chiens peu importe le CRUD peut prendre n’importe quel objet et avec une id faire l’une des quatre actions de bases. L’ID est également important comme paramètre, car il permet de viser l’objet sur lequel on veut effectuer l’action. Le C permet de créer un Objet T. Le R permet d’afficher les données/les lire. Update permet de modifier les données de l’objet, cela peut être l’âge d’un homme, la taille… D pour delete afin de pouvoir supprimer l’objet T.

Mini tutoriel Relations

Tableau des conversions

Nom Explication Image
Association L’Association : Une association est lorsque A utilise B.
Héritage Lorsque l’objet B hérite de l’objet A cela signifie que B va prendre les variables de la classe Parent A puis va par la suite rajoute à celles-ci ses propres variables. On peut imaginer A comme étant un Parent et B un enfant qui va prendre des caractéristiques de ses parents A mais aussi avoir ses propres caractéristiques.
L’Implémentation Signifie que l’interface A implémente l’interface B
L’Agrégation Cela signifie que A est un attribut de la classe B mais qu’il n’a pas besoin d’être instancier du constructeur de B. Par exemple : Un appartement A qui contient des meubles B. Un appartement n’a pas besoin de meuble pour être un appartement.
La Composition Cela signifie que A est un attribut de la classe B qui doit être obligatoirement instancié dans le constructeur B. Par exemple : Un appartement A n’est pas un appartement sans un toit B. B doit être instancié lorsqu’on utilise A.

Tableau des associations

Nom Explication Image
L’association 1 à 1 La classe A est en relation 1 à 1 avec la classe B. Cela signifie que A doit devenir un attribut de B ou l’inverse.
L’association 1 à plusieurs La classe A à un attribut de type B dans sa classe.
L’association plusieurs à plusieurs Il faut créer une classe AB qui aura un attribut de chaque classe (un attribut de A et un attribut de B).
